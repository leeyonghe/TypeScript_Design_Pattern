# TypeScript Design Pattern (교육용)

- 객체 지향 설계 원칙
  1. 단일 책임 원칙
  2. 개방 폐쇄 원칙
  3. 리스코프 치환 원칙
  4. 인터페이스 분리 원칙
  5. 의존 역전 원칙

- 생성 패턴
  1. 싱글턴(Singleton)
  2. 빌더(Builder)
  3. 팩토리 메서드(Factory Method)
  4. 프로토타입(Prototype)
  5. 추상팩토리(Abstract Factory)

- 구조 패턴
  1. 어댑터(Adapter)
  2. 브리지(Bridge)
  3. 컴포지트(Composite)
  4. 데커레이터(Decorator)
  5. 퍼사드(Facade)
  6. 플라이웨이트(Flyweight)
  7. 프록시(Proxy)

- 행위 패턴
  1. 책임 연쇄(Chain of Responsibillity)
  2. 커맨드(Command)
  3. 인터프리터(Interpreter)
  4. 이터레이터(Iterator)
  5. 미디에이터(Mediator)
  6. 메멘토(Memento)
  7. 옵저버(Observer)
  8. 스테이트(State)
  9. 스트래티지(Strategy)
  10. 템플릿 메서드(Template Method)
  11. 비지터(Visitor)
